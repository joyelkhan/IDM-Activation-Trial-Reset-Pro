# Contributing Guidelines

We love your input! We want to make contributing to this project as easy and transparent as possible.

## Development Process

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Test thoroughly
4. Ensure code follows existing style
5. Update documentation as needed
6. Submit a pull request

## Code Style

### PowerShell Guidelines
- Use PascalCase for functions and classes
- Use camelCase for variables and parameters
- Include comprehensive comment-based help
- Add error handling for all operations
- Use approved verbs for functions

### Documentation
- Update README.md for new features
- Include examples for new parameters
- Add to CHANGELOG.md for significant changes

## Pull Request Process

1. Update documentation reflecting changes
2. Test on multiple Windows versions if possible
3. Ensure no breaking changes to existing functionality
4. PR will be reviewed within 48 hours

## Feature Requests

We welcome feature requests! Please open an issue with:
- Use case description
- Proposed implementation
- Potential impact

## Bug Reports

Please use the bug report template and include:
- Windows version and PowerShell version
- Steps to reproduce
- Expected vs actual behavior
- Error messages and logs

## Code of Conduct

### Our Pledge
We pledge to make participation in our project a harassment-free experience for everyone.

### Our Standards
- Using welcoming and inclusive language
- Being respectful of differing viewpoints
- Gracefully accepting constructive criticism
- Focusing on what is best for the community

### Enforcement
Instances of abusive behavior may be reported to the project team. All complaints will be reviewed and investigated.

## Testing Guidelines

Before submitting a PR, please test:
1. Fresh IDM installation
2. Already activated IDM
3. Trial expired IDM
4. Different Windows versions (if possible)
5. Both PowerShell 5.1 and 7.0+ (if possible)

## Documentation Standards

- Use clear, concise language
- Include code examples
- Add screenshots for UI changes
- Update all relevant documentation files
